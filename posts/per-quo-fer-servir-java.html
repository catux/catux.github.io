Per què no fer servir Java?

Escrit al 2005-02-19 00:00:00 per cpina

No sóc un expert de Java, però mirant una documentació de les llibreries Qt he vist uns enllaços relacionats amb rendiment i comparatives de Java prou interessants.

El primer de tots és A Java Critique, un article interessant i raonant coses del disseny de Java que no són (o poden no ser) totalment idònies.

També hi ha un PDF comparant alguns aspectes del rendiment de Java i Qt, des d'execució a claretat del codi.

La documentació de Qt també ens ensenya un cas d'ús de Qt (una important empresa Noruega que ha canviat de Java a Qt). Ja sabem que a Noruega solen estar prou conscienciats amb el tema de programari lliure.

Per últim, i més amb l'aspecte de llibertats, fa temps es va publicar un article d'en Richard Stallman: The Java Trap. Tracta sobre les llibertats de les llicències de Java i com un programa fet en Java no pot ser 100% lliure per dependència amb el Java mateix.

Tot això no treu que Java pugui tenir algunes coses bones, evidentment.
